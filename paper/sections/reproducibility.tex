\section{Reproducibility and Data Access}
\label{sec:reproducibility}

This appendix provides concrete run-level provenance for the reported
tables/figures and data-access constraints.

\subsection{Immutable Archive and Manifest}

An immutable reproducibility archive is available on Zenodo
\citep{eziz2026zenodoejorrepro}. It contains:
\begin{itemize}
\item analysis scripts used for the manuscript (\texttt{analysis/06\_moltbook\_only\_analysis.py},
\texttt{analysis/07\_reddit\_only\_analysis.py},
\texttt{analysis/08\_cross\_platform\_matched\_analysis.py});
\item manuscript-facing artifacts under \texttt{paper/figures/} and
\texttt{paper/tables/};
\item run manifests and run-scoped outputs under \texttt{outputs/.../run\_*};
\item sanitized instance-level derived feature tables under
\texttt{data\_features\_sanitized/};
\item a complete checksum manifest (\texttt{SHA256SUMS.txt}) and machine-readable
file manifest (\texttt{RELEASE\_MANIFEST.json}).
\end{itemize}
The file \texttt{MANUSCRIPT\_ARTIFACT\_PROVENANCE.csv} in the Zenodo archive maps each
manuscript figure/table file to its run-scoped source artifact.

\subsection{Manuscript Artifact Linkage}

Manuscript-facing figure/table assets in \texttt{paper/figures/} and
\texttt{paper/tables/} are linked to run outputs via
\texttt{MANUSCRIPT\_ARTIFACT\_PROVENANCE.csv} (SHA-256 based). For this draft, most files
match run-scoped outputs exactly; two artifacts are documented as presentation-level
derivatives from the same runs:
\begin{itemize}
\item \texttt{paper/figures/reply\_hazard\_moltbook.png} is linked to
\texttt{outputs/moltbook\_only/run\_20260206-145240Z/tables/reply\_hazard\_binned.csv};
\item \texttt{paper/tables/cross\_platform\_matched\_paired\_effects.csv} differs from
run CSV formatting only and is linked to
\texttt{outputs/cross\_platform\_matched/run\_20260206-182842Z/tables/paired\_effects.csv}.
\end{itemize}

\subsection{Data Access and Redistribution Limits}

The primary Moltbook archive is publicly available at Hugging Face
\citep{simulamet2026observatoryarchive}. Reddit raw exports are not redistributed
because of platform terms. The Zenodo package therefore provides only
run-scoped aggregate outputs, run manifests, and sanitized instance-level derived
tables (IDs/authors pseudonymized; no raw post/comment text fields redistributed).
